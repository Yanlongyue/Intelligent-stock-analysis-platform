# 后端启动故障排查

## ❌ 问题现象
```
[1]+  Exit 2                  nohup python3 real_data_backend.py > backend.log 2>&1
```

后端启动失败，退出码2。

## 🔍 排查步骤

### 步骤1：查看错误日志
```bash
cd /www/wwwroot/Intelligent-stock-analysis-platform
cat backend.log
```

### 步骤2：常见错误及修复

#### 错误1：缺少依赖包
```bash
# 安装依赖
pip3 install flask flask-cors akshare tushare
```

#### 错误2：Tushare token未设置
```bash
# 设置环境变量（临时）
export TUSHARE_TOKEN="your_token_here"

# 或检查 .env 文件是否存在
cat .env
```

#### 错误3：端口9000被占用
```bash
# 检查9000端口
netstat -tlnp | grep 9000

# 杀掉占用进程
kill -9 <PID>
```

#### 错误4：Python模块导入错误
```bash
# 直接运行看详细错误
python3 real_data_backend.py
```

### 步骤3：直接在终端运行测试（不调后台）

```bash
cd /www/wwwroot/Intelligent-stock-analysis-platform
python3 real_data_backend.py
```

这样能看到实时报错信息。

## ✅ 修复后重新启动

```bash
# 1. 杀掉残留进程
pkill -f real_data_backend

# 2. 清除日志
> backend.log

# 3. 重新启动
cd /www/wwwroot/Intelligent-stock-analysis-platform
nohup python3 real_data_backend.py > backend.log 2>&1 &

# 4. 检查状态
sleep 2
ps aux | grep real_data_backend | grep -v grep
netstat -tlnp | grep 9000
```
