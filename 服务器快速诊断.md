# 服务器快速诊断

## 🔍 立即执行这些命令

```bash
cd /www/wwwroot/Intelligent-stock-analysis-platform

# 1. 检查后端是否还在运行
echo "=== 后端进程 ==="
ps aux | grep real_data_backend | grep -v grep

# 2. 检查端口
echo ""
echo "=== 端口监听 ==="
netstat -tlnp | grep 9000

# 3. 查看后端日志（最后20行）
echo ""
echo "=== 后端日志 ==="
tail -20 backend.log

# 4. 测试关键API
echo ""
echo "=== API测试 ==="
curl -s http://localhost:9000/api/health
echo ""
curl -s http://localhost:9000/api/positions | head -c 100
```

## 🚀 如果后端挂了，一键重启

```bash
cd /www/wwwroot/Intelligent-stock-analysis-platform

# 杀掉旧进程
pkill -f real_data_backend
sleep 1

# 重新启动
nohup python3 stock_analysis_program/real_data_backend.py > backend.log 2>&1 &

# 等待启动
sleep 3

# 验证
netstat -tlnp | grep 9000
curl -s http://localhost:9000/api/health
```

## 📝 常见错误

如果日志显示 `ModuleNotFoundError`，需要安装依赖：

```bash
pip3 install flask flask-cors akshare tushare
```

如果显示数据库连接错误，检查MySQL是否运行：

```bash
systemctl status mysqld
# 或
bt default | grep -A 3 MySQL
```
