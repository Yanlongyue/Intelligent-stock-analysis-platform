# 股票分析平台 - 服务器快速启动指南

## 🚀 一键启动命令

在服务器上按顺序执行以下命令：

```bash
# ========== 1. 启动后端API（端口9000） ==========
cd /www/wwwroot/Intelligent-stock-analysis-platform
nohup python3 real_data_backend.py > backend.log 2>&1 &

# 等待2秒，检查后端是否启动
sleep 2
netstat -tlnp | grep 9000

# 测试API是否正常
curl -s http://localhost:9000/api/analyze_all | python3 -c "import sys,json; d=json.load(sys.stdin); print('✅ 后端正常，股票数:', len(d.get('stocks', [])))"


# ========== 2. 启动前端（端口8889，避开宝塔8888） ==========
# 先杀掉可能残留的旧进程
pkill -f "http.server"

# 启动前端
nohup python3 -m http.server 8889 --bind 0.0.0.0 > frontend.log 2>&1 &

# 检查前端是否启动
sleep 1
netstat -tlnp | grep 8889


# ========== 3. 验证服务状态 ==========
echo "=== 服务状态检查 ==="
ps aux | grep python3 | grep -E "http.server|real_data_backend" | grep -v grep
echo ""
echo "=== 端口监听 ==="
netstat -tlnp | grep -E "8889|9000"
```

## 📝 常见问题

### 端口被占用？
```bash
# 查看占用端口的进程
netstat -tlnp | grep -E "8889|9000"

# 杀掉指定进程（替换PID）
kill -9 <PID>
```

### 后端启动失败？
```bash
# 查看错误日志
cat /www/wwwroot/Intelligent-stock-analysis-platform/backend.log
```

### 宝塔防火墙拦截？
在宝塔面板 → 安全 → 防火墙中放行：
- TCP 8889
- TCP 9000

## 🔒 阿里云安全组

确保阿里云安全组已放行：
- 入方向：TCP 8889/0.0.0.0/0
- 入方向：TCP 9000/0.0.0.0/0
