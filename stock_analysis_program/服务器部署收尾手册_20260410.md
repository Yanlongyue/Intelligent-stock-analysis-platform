## 🔧 第一步：安装/确认 MySQL

### 方式 A：通过宝塔面板安装（推荐）

1. 登录宝塔面板 → **软件商店**
2. 搜索 **MySQL** → 点击 **安装**
3. 推荐版本：**MySQL 5.7** 或 **8.0**
4. 设置 root 密码（记住这个密码！）

### 方式 B：命令行检查是否已安装

```bash
# 检查 MySQL 是否已安装
mysql --version

# 如果未安装，用 yum 安装
yum install -y mysql-server mysql
# 或者（CentOS 8+ / Alibaba Linux 3）
yum install -y mysql8 mysql-server
```

### 验证 MySQL 运行状态

```bash
# 查看服务状态
systemctl status mysqld

# 启动 MySQL（如未运行）
systemctl start mysqld

# 设置开机自启
systemctl enable mysqld
```

> ⚠️ **如果服务器资源紧张（1核2G以下），也可以不装 MySQL**，系统会自动降级为 JSON 文件模式，不影响核心功能。

## 🗄️ 第二步：创建数据库和用户

### 方法一：通过宝塔 phpMyAdmin 操作（最简单）

1. 宝塔面板 → **数据库** → **添加数据库**
   - 数据库名：`stock_analysis`
   - 用户名：`stock_user`
   - 密码：**自定义一个强密码**（例如 `St0ck@2026!`）
   - 访问权限：**本地服务器**（选 127.0.0.1）
2. 点击 **phpMyAdmin** 进入数据库管理界面
3. 点击 **导入** → 选择上传 `init_db.sql` 文件
4. 执行成功后应看到 6 张表 + 3 条默认持仓数据

### 方法二：命令行操作

```bash
# 1. 使用 root 登录 MySQL
mysql -u root -p

# 2. 在 MySQL 中依次执行：

-- 创建数据库
CREATE DATABASE IF NOT EXISTS `stock_analysis`
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- 创建专用用户（请把密码改成你自己的！）
CREATE USER IF NOT EXISTS 'stock_user'@'127.0.0.1'
  IDENTIFIED BY '你的数据库密码';

-- 授权
GRANT ALL PRIVILEGES ON `stock_analysis`.* TO 'stock_user'@'127.0.0.1';
FLUSH PRIVILEGES;

-- 切换到目标数据库
USE stock_analysis;

-- 建表（直接复制 init_db.sql 内容粘贴执行，或用 source 命令）
SOURCE /www/wwwroot/Intelligent-stock-analysis-platform/stock_analysis_program/init_db.sql;

# 3. 退出
exit;
```

## 🛡️ 第七步：（可选但强烈建议）配置宝塔守护进程

让后端开机自启、崩溃自动重启：

### 方法 A：使用宝塔 Python 项目管理器

1. 宝塔面板 → **软件商店** → **Python 项目管理器** → **安装**
2. 点击 **添加项目**：
   | 字段 | 值 |
   |------|-----|
   | 项目名称 | `stock-analysis-api` |
   | 项目路径 | `/www/wwwroot/Intelligent-stock-analysis-platform/stock_analysis_program` |
   | 启动文件 | `real_data_backend.py` |
   | 运行端口 | `9000` |
   | 开机自启 | ✅ 开启 |
   | 环境变量 | `MYSQL_PASSWORD=你的密码` `TUSHARE_TOKEN=你的token` |

### 方法 B：使用 systemd 服务（更可靠）

```bash
# 创建服务文件
cat > /etc/systemd/system/stock-api.service << 'EOF'
[Unit]
Description=Stock Analysis API Backend
After=network.target mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/www/wwwroot/Intelligent-stock-analysis-platform/stock_analysis_program
ExecStart=/usr/bin/python3 real_data_backend.py
Restart=always
RestartSec=5
Environment="MYSQL_HOST=127.0.0.1"
Environment="MYSQL_PORT=3306"
Environment="MYSQL_USER=root"
Environment="MYSQL_PASSWORD=你的数据库密码"
Environment="MYSQL_DATABASE=stock_analysis"
Environment="TUSHARE_TOKEN=你的tushare_token"

[Install]
WantedBy=multi-user.target
EOF

# 启用服务
systemctl daemon-reload
systemctl enable stock-api
systemctl start stock-api
systemctl status stock-api
```

## 🐛 常见问题排查

### ❌ "pymysql 不存在" 错误
```bash
pip3 install pymysql
# 确认用的是同一个 Python3
which python3 && python3 -c "import pymysql"
```

### ❌ "Access denied for user" 数据库连接失败
```bash
# 检查用户名密码是否正确
mysql -u stock_user -p -h 127.0.0.1
# 检查用户权限
mysql -u root -e "SHOW GRANTS FOR 'stock_user'@'127.0.0.1';"
```

### ❌ "/api/db_stats" 返回空或报错
说明 MySQL 未正确连接。检查：
1. MySQL 服务是否运行：`systemctl status mysqld`
2. 环境变量是否正确：`env | grep MYSQL`
3. 查看 `backend.log` 中的错误信息：`tail -100 backend.log`
4. **降级模式**：即使 MySQL 连不上，系统会自动降级为 JSON 模式继续运行

### ❌ 端口 9000 被占用
```bash
# 找出谁占用了
lsof -i :9000
# 杀掉旧进程
kill -9 <PID>
```

### ❌ Nginx 代理超时
编辑 nginx 配置文件，将 `proxy_read_timeout` 从 60s 改为 **120s**：
```nginx
location /api/ {
    proxy_pass http://127.0.0.1:9000;
    ...
    proxy_read_timeout 120s;  # positions 接口需要较长时间获取实时数据
}
```

### ❌ 持仓数据丢失或不显示
```bash
# 检查 JSON 备份文件
cat data/positions.json
# 或查看 MySQL 表
curl -s http://127.0.0.1:9000/api/positions
```
