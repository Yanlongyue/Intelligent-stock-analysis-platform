# 🌐 股票分析系统网络访问指南

本文档详细说明如何在其他电脑上访问您的股票分析系统。

## 🚀 快速开始

### 最简单的方法（推荐）

```bash
# 1. 进入项目目录
cd /Users/yandada/WorkBuddy/Claw/stock_analysis_program

# 2. 运行网络访问启动脚本
./启动网络访问系统.sh
```

脚本会自动：
1. 启动API后端服务（端口9000）
2. 启动Web界面服务器（端口8888）
3. 显示您的局域网IP地址
4. 提供访问链接

### 访问地址

脚本运行后会显示类似以下信息：

```
📍 您的局域网IP地址: 192.168.1.100
🌐 您的公网IP地址: 123.45.67.89
```

**在其他电脑上访问：**
- **Web界面**: `http://192.168.1.100:8888/web_interface_enhanced.html`
- **API服务**: `http://192.168.1.100:9000/api/health`

## 🔧 详细配置说明

### 方法一：同一局域网内访问

#### 要求：
1. 两台电脑连接到同一个WiFi或路由器
2. 防火墙允许端口8888和9000

#### 步骤：

1. **在您的Mac上：**
   ```bash
   # 获取您的IP地址
   ifconfig | grep "inet " | grep -v 127.0.0.1
   ```
   输出类似：`inet 192.168.1.100 netmask 0xffffff00 broadcast 192.168.1.255`

2. **启动系统：**
   ```bash
   ./启动网络访问系统.sh
   ```

3. **在其他电脑上：**
   - 打开任何现代浏览器（Chrome、Firefox、Edge等）
   - 输入地址：`http://192.168.1.100:8888/web_interface_enhanced.html`
   - 按回车访问

### 方法二：外网访问（通过互联网）

#### 要求：
1. 路由器支持端口转发
2. 有公网IP地址（或使用内网穿透）

#### 步骤：

1. **路由器设置端口转发：**
   - 登录路由器管理界面（通常是192.168.1.1或192.168.0.1）
   - 找到"端口转发"或"虚拟服务器"设置
   - 添加两条规则：
     ```
     规则1: 外部端口8888 → 内部IP:192.168.1.100 内部端口:8888
     规则2: 外部端口9000 → 内部IP:192.168.1.100 内部端口:9000
     ```
   - 保存设置

2. **获取公网IP：**
   - 运行启动脚本会显示公网IP
   - 或访问：https://www.ip138.com

3. **在其他电脑上访问：**
   - 输入地址：`http://您的公网IP:8888/web_interface_enhanced.html`
   - 例如：`http://123.45.67.89:8888/web_interface_enhanced.html`

### 方法三：使用内网穿透工具（无公网IP）

如果您的路由器不支持端口转发或没有公网IP，可以使用内网穿透工具：

1. **推荐工具：**
   - **ngrok**（简单易用）：https://ngrok.com
   - **frp**（免费开源）：https://github.com/fatedier/frp
   - **花生壳**（国内服务）：https://hsk.oray.com

2. **使用ngrok示例：**
   ```bash
   # 安装ngrok
   brew install ngrok  # macOS
   
   # 启动Web服务器
   python3 run_for_network_access.py
   
   # 在另一个终端启动ngrok
   ngrok http 8888
   ```
   ngrok会生成一个公网地址，如：`https://abc123.ngrok.io`

## 🛠️ 防火墙配置

### macOS防火墙设置

如果其他电脑无法访问，请检查防火墙：

1. **打开系统偏好设置** → **安全性与隐私** → **防火墙**
2. **点击锁图标解锁**（需要管理员密码）
3. **点击"防火墙选项..."**
4. **确保允许以下端口的入站连接：**
   - 端口8888 (HTTP)
   - 端口9000 (API)

### 使用命令行检查防火墙

```bash
# 检查防火墙状态
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate

# 临时禁用防火墙（不推荐）
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off

# 启用防火墙
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate on
```

## 🔍 故障排除

### 问题1：无法访问页面
**症状**：浏览器显示"无法连接到服务器"或"连接超时"

**解决方案：**
1. **检查IP地址是否正确**
   ```bash
   # 在您的Mac上重新获取IP
   ifconfig | grep "inet " | grep -v 127.0.0.1
   ```

2. **检查服务器是否运行**
   ```bash
   # 查看端口是否在监听
   lsof -i :8888
   lsof -i :9000
   ```

3. **检查防火墙**
   ```bash
   # 临时禁用防火墙测试
   sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off
   ```

### 问题2：页面显示但API不工作
**症状**：Web界面能打开，但五维度分析显示"API未连接"

**解决方案：**
1. **检查API服务是否运行**
   ```bash
   curl http://localhost:9000/api/health
   ```

2. **检查网络连接**
   ```bash
   ping 192.168.1.100  # 替换为您的IP
   ```

3. **重启API服务**
   ```bash
   python3 algorithm_backend_network.py
   ```

### 问题3：连接速度慢
**解决方案：**
1. 确保两台电脑在同一网络（避免跨路由器）
2. 关闭其他占用带宽的应用
3. 使用有线连接代替WiFi（如果可能）

## 🎯 最佳实践

### 1. **固定IP地址**
为避免IP地址变化导致无法访问，建议在路由器中为您的Mac设置固定IP：

1. 登录路由器管理界面
2. 找到DHCP设置或静态IP分配
3. 将您的Mac的MAC地址绑定到固定IP（如192.168.1.100）

### 2. **使用域名访问**
如果您有域名，可以设置DNS解析：
- A记录指向您的公网IP
- 使用子域名如：`stock.yourdomain.com`

### 3. **安全考虑**
- 仅在家用网络或可信网络中使用
- 考虑添加简单的HTTP Basic认证
- 定期更新系统

### 4. **性能优化**
- 关闭不需要的后台应用
- 使用5GHz WiFi频段
- 确保足够的RAM和CPU资源

## 📞 技术支持

### 常见问题快速解决

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无法连接 | 防火墙阻止 | 检查防火墙设置 |
