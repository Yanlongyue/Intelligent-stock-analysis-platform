## 🔧 如何更新持仓

### 1. 修改持仓配置文件
编辑文件：`config/position_config.py`

```python
# 在 PositionConfig 类中修改：
USER_POSITIONS = {
    "601868.SH": 400,   # 股票代码: 持仓数量
    "002506.SZ": 400,
    "600821.SH": 600,
    # 可以添加或删除股票
}
```

### 2. 添加新股票
如果要添加新股票，需要同时更新：
- `USER_POSITIONS`：持仓数量
- `POSITION_TYPES`：股票类型（value/growth/concept）
- `RISK_TOLERANCE`：风险等级（low/medium/high）
- `TARGET_POSITION_RATIOS`：目标仓位比例

## 🎯 七步法分析流程

每次运行 `./start.sh analyze`（或兼容旧入口 `python3 analyze_position_stocks.py`）会执行：

1. **第0步：强制长期记忆检索**
   - 自动读取分析规则和历史经验
   - 验证分析框架一致性

2. **第1步：数据获取**
   - 使用Tushare Pro API获取实时行情数据
   - 获取财务数据和市场指标

3. **第2步：深度复盘**
   - 对比昨日预测与今日实际走势
   - 分析预测误差和原因

4. **第3步：明日预测**
   - 生成明日小时级预测（4个时间段）
   - 提供涨跌幅预测和操作建议

5. **第4步：投资计划**
   - 制定具体仓位管理方案
   - 设置止盈止损价位

6. **第5步：风险控制**
   - 明确操作纪律和风险等级
   - 提供国际局势监控要点

7. **第6步：其他推荐**
   - 推荐2-3个有潜力的其他股票
   - 提供详细的投资理由和策略

## ⚠️ 重要注意事项

### 1. 数据准确性
- **核心原则**：数据错了，分析再多都是错的
- 每次分析前都会自动验证数据准确性
- 使用Tushare Pro API确保数据真实性

### 2. 对比方法
- 使用 **昨天预测的股价** 来对比 **今天的实际股价**
- 确保分析的一致性和可验证性

### 3. 样式布局
- 保持与20260401_v2模板完全相同的结构和排版
- 确保格式统一，便于阅读和比较

### 4. 风险控制
- 严格执行止损纪律：高风险股3%，中风险股5%
- 建议仓位上限：总资金的30%-50%
- 分批操作：买入30%/40%/30%，卖出30%/30%/40%

## 🔍 常见问题

### Q1: 如何添加股票持仓？
A: 修改 `config/position_config.py` 中的 `USER_POSITIONS` 字典，然后运行 `python3 test_position_config.py` 验证。

### Q2: 如何获取实时价格数据？
A: 先在终端执行 `export TUSHARE_TOKEN=您的Token`，程序会自动使用 API 获取数据。

### Q3: 分析结果在哪里查看？
A: 程序会生成两种格式的报告：
   - Markdown：`reports/markdown/` 目录下
   - HTML：`reports/html/` 目录下（建议用浏览器打开）

### Q4: 如何调整分析模型参数？
A: 修改 `config/settings.py` 中的权重参数，或使用长期记忆中记录的最新优化方案。

### Q5: 分析报告有错误怎么办？
A: 检查 `logs/quality.log` 查看质量检查报告，根据建议修复问题。
