## 🚀 方法1：端口转发（最稳定，推荐）

### 核心原理
在路由器上设置规则，将公网请求转发到您的Mac。

### 详细步骤

#### 步骤1：获取网络信息
```bash
# 在您的Mac上获取局域网IP
ifconfig | grep "inet " | grep -v 127.0.0.1

# 获取公网IP
curl ifconfig.me
# 或
curl ipinfo.io/ip
```

#### 步骤2：登录路由器
1. **打开浏览器**，输入路由器管理地址：
   - 常见地址：`192.168.1.1`、`192.168.0.1`、`192.168.31.1`
   - 用户名/密码：通常在路由器背面标签上

2. **找到端口转发设置**：
   - 可能叫：端口转发、虚拟服务器、NAT设置、应用规则
   - 通常在：高级设置 → 安全 → 端口转发

#### 步骤3：配置转发规则
添加以下两条规则：

**规则1 - Web服务器**：
```
服务名称: StockWeb
协议类型: TCP
外部端口: 8888
内部IP地址: 192.168.1.59
内部端口: 8888
```

**规则2 - API服务**：
```
服务名称: StockAPI
协议类型: TCP
外部端口: 9000
内部IP地址: 192.168.1.59
内部端口: 9000
```

#### 步骤4：保存并重启
1. 保存设置
2. 重启路由器（重要！）
3. 等待2-3分钟路由器重启完成

#### 步骤5：测试访问
```bash
# 测试Web服务
curl -I http://[您的公网IP]:8888/web_interface_enhanced.html

# 测试API服务
curl http://[您的公网IP]:9000/api/health
```

#### 访问地址
```
Web界面: http://[您的公网IP]:8888/web_interface_enhanced.html
API服务: http://[您的公网IP]:9000/api/health
```

### 注意事项
✅ **优点**：
- 完全免费
- 最稳定可靠
- 延迟最低
- 长期可用

⚠️ **缺点**：
- 需要路由器配置权限
- 公网IP可能变化（可申请固定IP）
- 需要重启路由器

## 🌀 方法2：ZeroTier（虚拟局域网，免费）

### 核心原理
创建虚拟局域网，让所有设备像在同一个网络。

### 详细步骤

#### 步骤1：注册ZeroTier
1. 访问 https://my.zerotier.com
2. 注册免费账号
3. 登录控制台

#### 步骤2：创建网络
1. 点击"Create a Network"
2. 记下网络ID（16位字母数字）
3. 网络名称：`Stock-Analysis-Network`

#### 步骤3：在Mac上安装
```bash
# 使用Homebrew安装
brew install zerotier-one

# 启动服务
sudo zerotier-one -d

# 加入网络
sudo zerotier-cli join [您的网络ID]

# 查看状态
sudo zerotier-cli status
```

#### 步骤4：授权设备
1. 回到ZeroTier控制台
2. 在"Members"页面找到您的Mac
3. 勾选"Auth"复选框授权

#### 步骤5：获取ZeroTier IP
```bash
# 查看分配的IP
sudo zerotier-cli listnetworks
```
会显示类似：`2001:19f0:...` 或 `10.147.20.x`

#### 步骤6：在其他设备上安装
**Windows**：
1. 下载 https://www.zerotier.com/download/
2. 安装并加入同一网络ID
3. 在控制台授权

**iPhone/iPad**：
1. App Store搜索"ZeroTier"
2. 安装并加入网络
3. 在控制台授权

**Android**：
1. Google Play搜索"ZeroTier"
2. 安装并加入网络
3. 在控制台授权

#### 访问地址
```
Web界面: http://[ZeroTier IP]:8888/web_interface_enhanced.html
API服务: http://[ZeroTier IP]:9000/api/health
```

### 注意事项
✅ **优点**：
- 完全免费
- 支持所有设备
- 自动加密
- 无需路由器配置

⚠️ **缺点**：

## ☁️ 方法4：Cloudflare Tunnel（永久域名）

### 核心原理
通过Cloudflare创建安全隧道，获得永久域名。

### 详细步骤

#### 步骤1：注册Cloudflare
1. 访问 https://dash.cloudflare.com
2. 注册免费账号
3. 添加域名（可选，不是必须）

#### 步骤2：安装cloudflared
```bash
# 使用Homebrew安装
brew install cloudflared

# 登录
cloudflared tunnel login
# 会打开浏览器授权
```

#### 步骤3：创建隧道
```bash
# 创建隧道
cloudflared tunnel create stock-analysis

# 会显示隧道ID和证书文件位置
```

#### 步骤4：配置隧道
创建配置文件 `~/.cloudflared/config.yml`：
```yaml
tunnel: [您的隧道ID]
credentials-file: /Users/yandada/.cloudflared/[隧道ID].json

ingress:
  - hostname: stock-analysis.example.com
    service: http://localhost:8888
  - hostname: api.stock-analysis.example.com
    service: http://localhost:9000
  - service: http_status:404
```

#### 步骤5：创建DNS记录
```bash
# 创建DNS记录
cloudflared tunnel route dns stock-analysis stock-analysis
```

#### 步骤6：启动隧道
```bash
# 启动隧道
cloudflared tunnel run stock-analysis
```

#### 访问地址
```
Web界面: https://stock-analysis.trycloudflare.com
API服务: https://api.stock-analysis.trycloudflare.com
```

### 注意事项
✅ **优点**：
- 获得永久域名
- HTTPS自动加密
- 免费套餐足够
- 企业级安全

⚠️ **缺点**：
- 配置稍复杂
- 需要域名（或使用trycloudflare.com）

## 🚦 方法5：ngrok（临时测试）

### 核心原理
临时隧道，适合演示和测试。

### 详细步骤

#### 步骤1：安装ngrok
```bash
# 下载ngrok
curl -fsSL https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-darwin-amd64.zip -o ngrok.zip
unzip ngrok.zip
chmod +x ngrok

# 注册获取token
# 访问 https://dashboard.ngrok.com/signup
```

#### 步骤2：配置token
```bash
./ngrok config add-authtoken [您的token]
```

#### 步骤3：启动隧道
```bash
# 启动Web服务隧道
./ngrok http 8888

# 启动API服务隧道
# 需要两个终端，或使用配置文件
```

#### 访问地址
ngrok会显示类似：
```
Forwarding: https://abc123.ngrok.io -> http://localhost:8888
```

#### 配置文件方法
创建 `ngrok.yml`：
```yaml
version: "2"
authtoken: [您的token]
tunnels:
  web:
    proto: http
    addr: 8888
    host_header: "localhost:8888"
  api:
    proto: http
    addr: 9000
    host_header: "localhost:9000"
```

启动：
```bash
./ngrok start --config ngrok.yml --all
```

### 方案2：反向代理
```nginx
# Nginx配置示例
server {
    listen 443 ssl;
    server_name stock.yourcompany.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://192.168.1.59:8888;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /api/ {
        proxy_pass http://192.168.1.59:9000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 方案3：Docker部署
```dockerfile
# Dockerfile
FROM python:3.9-slim

WORKDIR /app
COPY . .

RUN pip install -r requirements.txt

EXPOSE 8888 9000

CMD ["bash", "start_real_data_system.sh"]
```

```bash

docker build -t stock-analysis .
```

## 🔧 故障排除

### 常见问题1：端口转发无效
```bash
# 检查端口是否开放
nmap -p 8888,9000 [您的公网IP]

# 检查防火墙
sudo lsof -i :8888
sudo lsof -i :9000

# 临时关闭防火墙测试
sudo systemctl stop firewalld  # CentOS
sudo ufw disable              # Ubuntu
```

### 常见问题2：连接超时
1. **检查服务是否运行**
   ```bash
   ps aux | grep python
   ```

2. **检查日志**
   ```bash
   # 查看Web服务日志
   tail -f web_server.log
   
   # 查看API服务日志
   tail -f api_server.log
   ```

3. **检查网络连通性**
   ```bash
   ping [目标IP]
   telnet [目标IP] 8888
   ```

### 常见问题3：SSL证书错误
如果使用HTTPS：
1. 申请免费证书：Let's Encrypt
2. 使用Cloudflare代理
3. 自签名证书（仅测试）

### 常见问题4：移动网络无法访问
某些运营商限制：
1. 使用HTTPS端口（443）
2. 使用非标准端口（8080、8443）
3. 使用VPN绕开限制
