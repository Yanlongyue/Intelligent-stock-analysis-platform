# 阿里云宝塔面板部署指南

> 目标：把 `real_data_backend.py` 部署到阿里云服务器，前端通过 `http://101.133.150.164:9000` 直接访问后端 API。

---

## 前置条件

- 阿里云服务器，已安装宝塔 Linux 面板 9.2.0
- 公网 IP：`101.133.150.164`
- 阿里云安全组已开放端口 `9000`
- 宝塔面板安全组同样需要放行（见下）

---

## 第一步：宝塔防火墙放行 9000 端口

1. 登录宝塔面板（通常是 `http://公网IP:8888`）
2. 左侧菜单 → **安全** → **系统防火墙**
3. 点击「添加规则」：
   - 端口：`9000`
   - 协议：`TCP`
   - 说明：`股票分析 API`
4. 保存

> ⚠️ 阿里云安全组已开放不够，宝塔自己的防火墙也要放行，两者都要做。

---

## 第二步：宝塔安装 Python 3.10+

1. 宝塔面板左侧菜单 → **软件商店**
2. 搜索「Python 项目管理器」，安装
3. 在 Python 项目管理器中，点击「安装 Python 版本」，选择 **3.10** 或 **3.11**
4. 安装完成后记下 Python 路径，通常是：
   ```
   /www/server/pyenv/versions/3.10.x/bin/python3
   ```
   或者直接 SSH 确认：
   ```bash
   python3 --version
   ```

---

## 第三步：上传代码到服务器

### 方式 A：宝塔文件管理器（推荐，不需要 SSH）

1. 宝塔面板 → **文件** → 进入 `/www/wwwroot/`
2. 点击「上传」，把整个 `stock_analysis_program/` 文件夹打包成 `.zip` 上传
3. 上传后在文件管理器里右键解压到 `/www/wwwroot/stock_analysis_program/`

### 方式 B：Git 拉取（需要 SSH）

```bash
cd /www/wwwroot
git clone https://github.com/Yanlongyue/Intelligent-stock-analysis-platform.git
# 代码就在 /www/wwwroot/Intelligent-stock-analysis-platform/stock_analysis_program/
```

---

## 第四步：安装 Python 依赖

在宝塔面板 → **终端**（或 SSH 进入服务器）执行：

```bash
cd /www/wwwroot/stock_analysis_program

# 安装依赖（跳过 sqlite3 和 python 版本行，它们不是 pip 包）
pip3 install tushare akshare pandas numpy jinja2 markdown markdown-it-py \
             pygments python-dateutil pytz schedule APScheduler openpyxl \
             fastapi "uvicorn[standard]" loguru prometheus-client pytest \
             python-dotenv pyyaml toml psutil tqdm
```

> 如果 `pip3` 提示找不到，换成 `pip` 或 `/www/server/pyenv/versions/3.10.x/bin/pip3`

---

## 第五步：配置 Tushare Token（可选，不配则走模拟数据）

```bash
# 临时设置（重启后失效）
export TUSHARE_TOKEN=你的Token

# 永久设置（推荐）
echo 'export TUSHARE_TOKEN=你的Token' >> ~/.bashrc
source ~/.bashrc
```

> 没有 Tushare Token 也能跑，后端会自动降级到 AkShare → 模拟数据。

## 第七步：用宝塔「Python 项目管理器」常驻运行

1. 宝塔面板 → **Python 项目管理器**
2. 点击「添加项目」：

   | 字段 | 填写内容 |
   |------|---------|
   | 项目名 | `stock-analysis-api` |
   | 项目路径 | `/www/wwwroot/stock_analysis_program` |
   | 启动文件 | `real_data_backend.py` |
   | Python 版本 | 选你安装的 3.10 / 3.11 |
   | 端口 | `9000` |
   | 开机自启 | ✅ 勾选 |

3. 点击「添加」，然后点「启动」
4. 项目状态变为「运行中」即可

> 如果宝塔 Python 管理器没有「启动文件」选项，可以改用「守护进程」方式（见下）。

## 常见问题排查

### 问题1：`curl` 能通，浏览器打不开
- 检查浏览器是否有混合内容拦截（GitHub Pages 是 HTTPS，调 HTTP 的 API 会被拦截）
- 解决：加 Nginx 反向代理 + Let's Encrypt 证书，升级到 HTTPS

### 问题2：`curl` 也不通
- 确认阿里云**安全组**和宝塔**防火墙**都已放行 9000
- 确认后端进程确实在运行：`ps aux | grep real_data_backend`

### 问题3：依赖安装失败
- 尝试换源：`pip3 install -i https://pypi.tuna.tsinghua.edu.cn/simple 包名`

### 问题4：GitHub Pages (HTTPS) 调 HTTP API 被浏览器拦截

这是**最可能遇到的问题**，见下面的「升级 HTTPS」章节。

---

## 升级方案：Nginx 反向代理 + HTTPS（绑定域名后做）

如果后续绑定了域名，可以用宝塔一键配：

1. 宝塔 → **网站** → 添加站点（域名填你的域名）
2. 宝塔 → **SSL** → 申请 Let's Encrypt 免费证书
3. 宝塔 Nginx 配置里加反向代理：
   ```nginx
   location / {
       proxy_pass http://127.0.0.1:9000;
       proxy_set_header Host $host;
       proxy_set_header X-Real-IP $remote_addr;
   }
   ```
4. 这样 `https://api.你的域名.com` 就稳定映射到 9000 端口
5. 最后只需把 `config.js` 里的地址改成 `https://api.你的域名.com` 再推一次
