# 服务器部署修复指南

## 问题描述
1. 五维算法界面显示"当前暂无可展示的真实分析结果"
2. PDF导出报告为空白

## 修复说明
已在本地修复 `RealAlgorithmEngine` 类，允许在真实数据不可用时回退到模拟分析，确保前端界面正常显示数据。

## 服务器操作步骤

### 1. 连接到服务器
```bash
ssh root@101.133.150.164
```

### 2. 进入项目目录
```bash
cd /www/wwwroot/Intelligent-stock-analysis-platform/stock_analysis_program
```

### 3. 拉取最新代码
```bash
git pull origin master
```

### 4. 停止当前后端进程
```bash
pkill -f "python3 real_data_backend.py"
```

### 5. 启动修复后的后端服务
```bash
nohup bash start_real_data_system.sh > backend.log 2>&1 &
```

### 6. 检查服务状态
```bash
netstat -tlnp | grep :9000
ps aux | grep real_data_backend.py
```

### 7. 验证修复效果
通过以下方式验证接口是否正常工作：

```bash
# 测试健康检查
curl http://localhost:9000/api/health

# 测试分析接口
curl http://localhost:9000/api/analyze_all | jq '.stocks | length'

# 查看日志
tail -f backend.log
```

## 预期效果
修复后，前端界面将：
1. 显示五维算法评分卡片（模拟数据）
2. PDF导出功能正常工作
3. 小时级预测数据正常显示

## 故障排除

### 1. 如果启动失败
检查Python依赖：
```bash
pip3 install -r requirements.txt
```

### 2. 如果接口返回错误
检查后端日志：
```bash
tail -50 backend.log
```

### 3. 如果仍然无数据
手动测试分析函数：
```bash
python3 -c "
from real_algorithm_engine import RealAlgorithmEngine
from real_data_provider import get_data_provider
provider = get_data_provider(use_real_data=True)
engine = RealAlgorithmEngine(provider)
result = engine.analyze_stock('601868.SH')
print('算法评分:', result.get('algorithm_scores'))
print('综合评分:', result.get('comprehensive_score'))
"
```
