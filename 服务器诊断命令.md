# 服务器诊断命令

## 检查后端日志

```bash
# 查看后端错误日志
cd /www/wwwroot/Intelligent-stock-analysis-platform
tail -50 backend.log

# 查看后端进程是否还在运行
ps aux | grep real_data_backend | grep -v grep

# 测试各个API端点
curl -s http://localhost:9000/api/health
curl -s http://localhost:9000/api/positions
curl -s http://localhost:9000/api/market_overview
curl -s http://localhost:9000/api/moneyflow_hsgt
```

## 如果后端挂了，重新启动

```bash
cd /www/wwwroot/Intelligent-stock-analysis-platform

# 杀掉旧进程
pkill -f real_data_backend

# 重新启动
nohup python3 stock_analysis_program/real_data_backend.py > backend.log 2>&1 &

# 等待启动
sleep 3

# 检查端口
netstat -tlnp | grep 9000

# 测试API
curl -s http://localhost:9000/api/health
```

## 检查前端配置是否正确更新

```bash
# 检查config.js是否更新
cat /www/wwwroot/Intelligent-stock-analysis-platform/stock_analysis_program/config.js | grep production

# 应该显示: production: 'http://101.133.150.164:9000',
```
