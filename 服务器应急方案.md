# 服务器公网访问应急方案

## 方案一：直接编辑宝塔防火墙配置

```bash
# 宝塔防火墙规则文件
echo "8889/tcp" >> /www/server/panel/data/port.pl
echo "9000/tcp" >> /www/server/panel/data/port.pl

# 重启宝塔面板
/etc/init.d/bt restart
```

## 方案二：用80端口启动（最省事）

既然firewalld没运行，直接用80端口（默认开放）：

```bash
cd /www/wwwroot/Intelligent-stock-analysis-platform

# 杀掉旧的
pkill -f "http.server"

# 用80端口启动（需要root权限）
nohup python3 -m http.server 80 --bind 0.0.0.0 > frontend.log 2>&1 &

# 确认
sleep 1
netstat -tlnp | grep :80
```

然后访问：`http://101.133.150.164/stock_analysis_program/web_interface_enhanced.html`

## 方案三：宝塔面板可视化操作

1. 浏览器打开：`http://101.133.150.164:8888`（宝塔面板）
2. 登录宝塔
3. 左侧"安全" → "防火墙"
4. 点击"添加端口"
   - 端口：8889
   - 备注：股票分析前端
5. 再添加
   - 端口：9000
   - 备注：股票分析API

## 方案四：阿里云安全组（必做）

无论上面哪个方案，都需要阿里云安全组放行：

登录阿里云控制台 → ECS → 安全组 → 配置规则 → 入方向 → 添加规则：

| 协议类型 | 端口范围 | 授权对象 | 描述 |
|---------|---------|---------|------|
| 自定义TCP | 8889 | 0.0.0.0/0 | 股票分析前端 |
| 自定义TCP | 9000 | 0.0.0.0/0 | 股票分析API |
| 自定义TCP | 80 | 0.0.0.0/0 | HTTP（如果用方案二）|

## 推荐操作顺序

1. **先用方案二**（80端口启动前端）- 最快
2. **同时配置方案四**（阿里云安全组）- 根本解决
3. **有余力再弄方案三**（宝塔可视化）- 最规范
